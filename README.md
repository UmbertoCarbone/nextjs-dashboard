
# Next.js Dashboard - Learning Project

## Italiano

Questo progetto è stato realizzato seguendo il tutorial ufficiale di Next.js:
[https://nextjs.org/learn/dashboard-app](https://nextjs.org/learn/dashboard-app)

Si tratta di una **financial dashboard** full-stack con home page pubblica, pagina di login, pagine protette da autenticazione e gestione completa delle fatture (creazione, modifica, eliminazione).

### Capitoli completati

| # | Capitolo | Descrizione |
|---|----------|-------------|
| 1 | **Getting Started** | Creazione del progetto Next.js, struttura delle cartelle e avvio del server di sviluppo locale. |
| 2 | **CSS Styling** | Diversi approcci per stilare un'app Next.js: CSS Modules, Tailwind CSS e `clsx` per classi condizionali. |
| 3 | **Optimizing Fonts and Images** | Ottimizzazione di font con `next/font` e immagini con il componente `next/image` per migliorare le performance. |
| 4 | **Creating Layouts and Pages** | Creazione di pagine e layout annidati tramite il file-system routing di Next.js (App Router). |
| 5 | **Navigating Between Pages** | Navigazione client-side con il componente `<Link>`, hook `usePathname` e prefetching automatico. |
| 6 | **Setting Up Your Database** | Configurazione di un database Postgres su Vercel e seed iniziale dei dati. |
| 7 | **Fetching Data** | Recupero dati direttamente nei Server Components con query SQL, evitando waterfall con `Promise.all`. |
| 8 | **Static and Dynamic Rendering** | Differenza tra rendering statico e dinamico, e come usare `unstable_noStore` per il rendering dinamico. |
| 9 | **Streaming** | Streaming con `loading.tsx`, `<Suspense>` e skeleton UI per migliorare l'esperienza utente durante il caricamento. |
| 10 | **Adding Search and Pagination** | Implementazione di ricerca e paginazione tramite URL Search Params con `useSearchParams`, `useRouter` e `usePathname`. |
| 11 | **Mutating Data** | Mutazione dei dati (crea, modifica, elimina fatture) con React Server Actions, validazione Zod, `revalidatePath` e `redirect`. |

## English

This project was built following the official Next.js tutorial:
[https://nextjs.org/learn/dashboard-app](https://nextjs.org/learn/dashboard-app)

It is a full-stack **financial dashboard** with a public home page, login page, authentication-protected pages, and full invoice management (create, edit, delete).

### Completed Chapters

| # | Chapter | Description |
|---|---------|-------------|
| 1 | **Getting Started** | Created the Next.js project, explored the folder structure, and started the local dev server. |
| 2 | **CSS Styling** | Explored different styling approaches in Next.js: CSS Modules, Tailwind CSS, and `clsx` for conditional classes. |
| 3 | **Optimizing Fonts and Images** | Optimized fonts with `next/font` and images with the `next/image` component for better performance. |
| 4 | **Creating Layouts and Pages** | Created nested pages and layouts using Next.js file-system routing (App Router). |
| 5 | **Navigating Between Pages** | Implemented client-side navigation with `<Link>`, the `usePathname` hook, and automatic prefetching. |
| 6 | **Setting Up Your Database** | Set up a Postgres database on Vercel and seeded it with initial data. |
| 7 | **Fetching Data** | Fetched data directly in Server Components using SQL queries, avoiding waterfalls with `Promise.all`. |
| 8 | **Static and Dynamic Rendering** | Learned the difference between static and dynamic rendering and how to opt into dynamic rendering. |
| 9 | **Streaming** | Implemented streaming with `loading.tsx`, `<Suspense>`, and skeleton UIs to improve perceived load performance. |
| 10 | **Adding Search and Pagination** | Built search and pagination using URL Search Params with `useSearchParams`, `useRouter`, and `usePathname`. |
| 11 | **Mutating Data** | Mutated data (create, edit, delete invoices) using React Server Actions, Zod validation, `revalidatePath`, and `redirect`. |
